# Contributing to the DiscoverabilityOps Standard

Thank you for your interest in improving the DiscoverabilityOps Standard.  
This document explains how to contribute changes, examples, or clarifications.

------------------------------------------------------------
## Ways to Contribute

You may contribute by:
- Suggesting clarifications
- Reporting inconsistencies or errors
- Adding examples in `/examples/`
- Providing implementation notes
- Proposing extensions or vertical-specific guidance
- Improving documentation clarity

------------------------------------------------------------
## Before Submitting Changes

Please review:

1. **README.md** (overview and repo structure)  
2. **spec/standard.md** (canonical specification)  
3. **CHANGELOG.md** (recent modifications)  
4. **GOVERNANCE.md** (decision-making process)

------------------------------------------------------------
## How to Submit Changes

### **1. Small fixes**
- Open a pull request directly (typos, formatting, simple clarifications).

### **2. Substantive changes**
(Changes to metrics, definitions, versioning, or schema.)
- Open a GitHub Issue first.
- Clearly describe:
  - the problem  
  - proposed change  
  - reason for change  
  - potential impact on implementations  

Maintainers may request examples, field data, or clarification.

------------------------------------------------------------
## Pull Request Guidelines

- Keep PRs focused (one change per PR).
- Update related documentation when applicable.
- Add entries to `CHANGELOG.md` if relevant.
- Use clear commit messages:
  - “Clarify ADR thresholds”
  - “Improve DOA schema example”
  - “Fix inconsistency in Fix-Levers list”

------------------------------------------------------------
## Style Guidelines

- Use clear, accessible language.
- Normative sections define the standard.
- Non-normative notes must be clearly marked.
- Use examples to show intent when appropriate.

------------------------------------------------------------
## Code of Conduct

All contributors must follow `CODE_OF_CONDUCT.md`.

------------------------------------------------------------
## Questions?

Open an Issue on GitHub with the label **question**.

We welcome community input and participation.
